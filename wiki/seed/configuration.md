# 模型参数配置与密钥分离

## 适用场景

当一个项目同时接入文本、图像、视频等多类模型接口时，默认参数往往会越来越多：

- 模型名称
- 温度
- 采样参数
- 分辨率
- 是否打水印
- 轮询间隔

如果这些参数直接散落在代码里，后续维护会很快变得困难。

## 推荐做法

建议将配置拆成两类：

### 1. 私密配置

用于不可提交的信息，例如：

- API Key
- 本地账号密钥

这类配置应：

- 优先支持环境变量
- 本地文件可作为兜底
- 默认不要提交到仓库

推荐命名：

- `config/seed_key.json`

### 2. 公共参数配置

用于可以共享的默认参数，例如：

- 模型名称
- 默认温度
- 默认尺寸
- 默认轮询超时

这类配置应：

- 允许提交到仓库
- 统一放在可版本管理的配置文件中
- 作为代码默认值的上层来源

推荐命名：

- `config/seed_config.json`

## 推荐收益

这样拆分后有几个明显好处：

- 安全信息与业务参数职责清晰
- 修改模型参数不需要改代码
- 不同接口可以共享一致的配置管理方式
- 团队协作时更容易同步默认策略

在 Seed 场景下，可以进一步固定约定：

- `seed_key.json` 只负责 API Key
- `seed_config.json` 只负责模型与请求默认参数

## 工程建议

如果项目接入多类 AIGC 接口，建议配置文件按能力分组，例如：

- `llm`
- `image`
- `video`

每个分组只放该能力真正关心的默认参数，避免混在一起难以维护。
