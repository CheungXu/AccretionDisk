# Sprout Web 工作台计划

## 目标
`sprout` 分两期交付：

1. 第一期先完成后端能力，包括项目导入、项目注册表、目录标准化、API、运行时日志、媒体访问、节点版本管理、热插拔重跑与测试验收。
2. 第二期在第一期测试完成并经确认后，再完成 `HTML/CSS/JS` 前端页面与交互。

## 代码分层
`agents/sprout/` 内部按职责拆分：

- `core/`：核心工作流与领域模型。
- `service/`：后端支撑代码，包括 API、项目导入、运行时、版本与媒体访问。
- `web/`：前端交互代码，第二期实施。
- `contracts/`：前后端共享契约。

分层约束：

- `core/` 不依赖 `service/` 或 `web/`。
- `service/` 依赖 `core/`，但不夹带前端页面逻辑。
- `web/` 只依赖接口契约，不直接读取磁盘目录。

## 一期范围
### 1. 目录与架构整理
- 为现有 `sprout` 建立 `core/`、`service/`、`web/` 子目录。
- 原有顶层模块保留兼容入口，新代码优先进入分层目录。

### 2. 多项目导入与管理
- 以 `data/sprout/node_test_case/` 为标准模板识别项目目录。
- 支持导入外部 AIGC 项目目录，并建立项目注册表。
- 支持原地引用与标准化复制两种导入模式。

### 3. 后端能力
- 提供项目列表、项目详情、节点详情、版本列表、日志读取、媒体访问等 API。
- 提供节点执行入口，支持从指定版本继续跑后续流程。
- 每次节点执行产出独立版本快照，并能切换激活版本。

### 4. 测试与文档
- 补充一期后端测试。
- 更新 `agents/sprout/readme.md` 与 `data/sprout/readme.md`。
- 在 `doc/` 中记录两期交付与一期后端架构。

## 二期范围
- 首页项目中心。
- 项目详情页。
- 节点详情页。
- 版本切换、重跑、视频播放与导入交互页面。

## 一期验收标准
- 能导入并注册 `sprout` 项目目录。
- 能通过 API 获取项目、节点、版本、日志与媒体信息。
- 能为节点创建新版本，并切换激活版本。
- 能基于某个版本继续执行后续节点。
- 测试通过后，再进入第二期。
