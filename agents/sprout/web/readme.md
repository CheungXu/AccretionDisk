# Sprout Web

`web/` 用于放置 `sprout` 工作台的前端交互代码，当前已经落地原生 `HTML/CSS/JS` 页面。

## 当前目录职责

- `pages/`：页面入口与页面脚本
- `components/`：页面渲染与通用 UI 拼装
- `services/`：前端接口调用封装
- `state/`：页面状态管理
- `assets/`：工作台样式与静态资源

## 当前能力

当前工作台页面已支持：

- 导入项目目录
- 查看项目列表与项目摘要
- 查看节点流程与节点详情
- 查看运行记录与日志
- 查看版本列表与版本内容
- 切换激活版本
- 触发节点重跑
- 预览图片与视频产物

## 依赖边界

- `web/` 只通过 `/api/*` 与后端交互
- `web/` 不直接读取磁盘目录
- 业务实现统一放在 `core/` 和 `service/`，前端不依赖核心实现模块
