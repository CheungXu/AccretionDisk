# Sprout Web

`web/` 用于放置 `sprout` 工作台的前端交互代码，当前已经落地原生 `HTML/CSS/JS` 页面。

## 当前目录职责

- `pages/`：页面入口与页面脚本
- `components/`：页面渲染与通用 UI 拼装
- `services/`：前端接口调用封装
- `state/`：页面状态管理
- `assets/`：工作台样式与静态资源

## 当前能力

当前工作台页面已支持：

- 通过原生目录选择器选择项目目录
- 支持将空目录初始化为空项目
- 查看项目列表与项目摘要
- 查看用户输入起始节点，并保存不同输入版本
- 查看与角色资产并行的脚本分镜节点及分镜图片预览
- 查看节点流程与节点详情
- 查看运行记录与日志
- 查看版本列表与版本内容
- 切换激活版本
- 触发节点重跑
- 预览图片与视频产物

## 当前工作流展示约定

- `用户输入` 是工作流起点，负责记录题材、时长、镜头数、画幅、风格、受众和分镜文本
- `脚本分镜` 是只读展示节点，不单独执行，展示内容直接跟随 `用户输入` 的激活版本
- `角色资产` 之后仍沿主链路进入提示词准备、视频生成、执行卡、导出与最终成片

## 当前镜头展示约定

- 画面描述读取 `visual_description`
- 首帧提示词读取 `keyframe_prompt`
- 视频提示词读取 `video_prompt`
- 分镜图片优先展示 `shot_keyframe`

## 依赖边界

- `web/` 只通过 `/api/*` 与后端交互
- `web/` 不直接读取磁盘目录
- 业务实现统一放在 `core/` 和 `service/`，前端不依赖核心实现模块
