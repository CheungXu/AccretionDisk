# 目录说明

| 目录                | 说明                                               |
|---------------------|----------------------------------------------------|
| `config/`           | 配置文件目录，保存各类配置信息                     |
| `doc/`              | 日常工作文档，包含分析报告、工作记录等，更偏工作的细节信息，目录结构`doc/YYYYMMDD/`       |
| `wiki/`             | 项目wiki知识库，不同于日常流水账，这里记录的是最有价值、最精华、最能帮助他人的知识，整体关于项目的信息写在`wiki/`下，具体每个agent的专有知识写在`wiki/{agent_name}/`下    |
| `agents/`           | agent目录，存放不同智能体代码，目录结构`agents/{agent_name}`                    |
| `data/`             | 数据目录，存放相关数据，按照不同agent划分，目录结构`data/{agent_name}`            |
| `skills/`           | 封装好的技能，每个技能一个文件夹，例如`skills/add_xxx/`   |
| `tmp/`              | 只用于存放临时文件、脚本，定期会清理`|
| `module/`           | 模块代码目录，存放独立基础模块代码，目录结构`code/{project_name}`|
| `script/`           | 常用脚本目录，保存非Agent本身的一些处理脚本，包括数据处理、任务启动等等|

# 工作原则
1. **工作语言为中文。**
2. **项目职责**：本项目`AccretionDisk`，主要工作为构建一下AIGC智能体，包括支持文本、图像、视频及多模态的AI创意实现。
3. **工作目录管理**：
   - 目录请严格按以上目录维护，工作中只能在以上目录中新增修改或删除文件，不在主目录下进行文件处理，如需新增目录，先进行询问，确认后新增。
   - 每次任务按理解和规划，合理管理各目录下的文件增删改，保持良好的项目架构。
   - 为避免过多信息堆积到最上层，代码下每一级目录可以有一个readme，但要按照你的理解合理规划，平衡单文件过于聚集和多文件信息过于分散。
4. **工作知识沉淀**：
   - 不同文档的区别：
      - `code/{agent_name}/readme.md`: 项目代码中的readme，主要**面向开发者**，提供使用方法、代码架构、配置信息等开发细节信息，目的是帮助开发者快速、详细了解代码，实现更加高效准确的开发。
      - `doc/{YYYYMMDD}/xxx.md`: 项目的工作信息，主要**面向项目管理**，隐去具体的技术和代码细节，提供项目背景、方案架构、实施规划流程、进度等信息，目的是让不懂技术管理者快速了解当前项目状态，快速参与规划与决策。
      - `wiki/`: 知识库，前面二者最主要的目的都是让相关人员快速了解本项目信息，加速迭代。但知识库是抽象出可跨项目复用的知识，目的是帮助**其他项目**快速了解通用知识、复用经验，以推动整体工作效率的提升。
   - 每次完成工作后，思考：
      - 项目代码改动后，是否有对应更新readme。
      - 是否有信息需要记录到doc，以为后续该任务的继续处理提供细节支持。
      - 是否有核心知识需要记录到wiki中，以为其他任务提供帮助。
