# Seed API 工作记录

## 工作背景

本轮工作目标是为项目补充字节 Seed 的基础接口能力，覆盖文本理解、生图、视频生成三条主链路，为后续智能体调用提供统一底层能力。

## 本次进展

本次已完成：

- Seed 文本与多模态理解接口接入
- Seed 生图接口接入
- Seed 视频生成任务接口接入
- API Key 的环境变量与配置文件双通道读取
- Seed 公共参数配置文件接入
- API Key 私密配置文件重命名为 `config/seed_key.json`
- 生图结果本地保存能力
- 视频结果本地保存能力
- 组图不足时的差量重试机制

## 当前状态

当前模块已经具备可用的基础能力，能够支撑：

- 文本生成与多模态理解
- 文生图与图生图
- 文生视频与图生视频任务创建
- 单图与组图生成
- 图片与视频结果本地落盘
- 通过统一配置文件维护默认模型参数

当前配置分层已经明确：

- `config/seed_key.json`：本地私密密钥配置
- `config/seed_config.json`：可提交的公共默认参数配置

整体接入工作已达到可继续向上层业务复用的状态。

## 风险与问题

当前主要风险集中在 `image_to_group` 场景：

- 单参考图到组图的生成稳定性不足
- 某些任务虽然请求多张图片，但接口只返回部分结果
- 该现象会影响需要完整组图输出的业务场景

目前判断，该问题更偏向上游生成稳定性或提示词约束不足，而不是本地封装错误。

视频链路当前状态：

- 图生视频真实任务已成功创建、查询并完成
- 视频结果已可下载到本地
- 当前未发现类似组图那样的稳定性问题

## 已采取措施

为降低业务误判风险，当前已采取以下措施：

- 组图任务按剩余目标张数自动重试
- 若多轮后仍未补齐目标数量，则直接报错
- 保留测试结果与问题记录，便于后续持续观察
- 视频任务支持真实轮询查询与本地下载
- 默认模型参数改为集中配置，降低后续维护成本
- 密钥配置与公共参数配置职责进一步拆分
- 视频配置已支持默认控制有声/无声输出

## 对项目的影响

正面影响：

- Seed 基础能力已可直接复用
- 后续接入上层 Agent 时，重复实现成本会明显下降
- 视频生成链路已具备基本可交付能力

待关注影响：

- 若业务场景依赖稳定的整组图片输出，仍需预留额外兜底策略

## 后续事项

- 持续观察 `image_to_group` 的稳定性
- 评估是否需要针对组图场景优化提示词模板
- 评估是否需要补充更明确的失败兜底策略
- 后续补充视频链路的更多任务场景与说明文档
- 视后续接入情况继续扩展 `config/seed_config.json` 的参数粒度
- 后续视协作需要再决定是否补充更多配置字段说明
